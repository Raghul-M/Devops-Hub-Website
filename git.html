Git and GitHub - Learning Notes

Introduction

1. Git:

Git is a distributed version control system that allows developers to track changes in their codebase and collaborate with others. It provides a way to manage and track different versions of a project, making it easier to revert to previous versions if necessary.

2. GitHub:

GitHub is a web-based platform that hosts Git repositories and offers additional features such as issue tracking, pull requests, and code reviews. It facilitates collaboration between developers by allowing them to work on the same codebase using branches and forks.

Repositories: A repository is a centralized location where you store and manage all the files and version history of your project, enabling collaboration and tracking of changes over time.

Git Commands

Git config:

The git config command is used to configure Git settings on your computer. These configurations can be global, per user, or per project/repository.

$ git config --global user.name "Your Name"
$ git config --global user.email "your@email.com"

Git init:

$ git init

Git Clone:

$ git clone

Git add:

$ git add   [Or]
$ git add .  # Adds all the files to staging area

Git commit:

$ git commit -m "Commit message"

Git status:

$ git status

Git Log:

$ git log

Git pull:

$ git pull origin

Git push:

$ git push origin

Git branch:

$ git branch  # Lists branches
$ git branch feature  # Creates a new feature branch
$ git checkout -b feature  # Creates and switches to the new feature branch

Git checkout:

$ git checkout

Git Merge:

$ git merge   # Merges feature branch to main

Git fetch:

$ git fetch --all

Git remote:

$ git remote -v  # View remote repositories

Git reset:

$ git reset   # Unstages a file
$ git reset   # Resets to a specific commit

Git diff:

$ git diff  # Shows the differences between changes

Git stash:

$ git stash  # Stashes changes for later
$ git stash pop  # Applies stashed changes
$ git stash clear  # Clears all stashes

Git remote add:

$ git remote add

Git remote remove:

$ git remote remove

Git help:

$ git help

Git Squash:

$ git rebase -i   # Interactive rebase to squash commits

Git Branching Strategy

Common Git branches include:

  • master/main - the main branch of your source code
  • feature - used for new features
  • release - for preparing a new production release
  • hotfix - for critical bug fixes in production

Git Authentication for Cloning

  • HTTPS: Uses a password or personal access token
  • SSH: Uses a public key (generate with ssh-keygen -t rsa)

Commonly Used Git Commands


$ git init  # Initializes a new Git repository, creates the .git directory to track version history
$ git add   # Adds file(s) to the staging area
$ git status  # Shows the current status of changes in the repository
$ git diff  # Displays the differences between changes in the working directory and staged changes
$ git commit -m "Message"  # Commits changes with a descriptive message
$ git log  # Displays the commit history
$ git reset --hard   # Resets the repository to a specific commit, discarding all changes after it
$ git push  # Pushes local commits to a remote repository
$ git clone   # Clones a remote repository to your local machine
$ git remote -v  # Shows remote repositories and their URLs
$ git remote add    # Adds a new remote repository with a specified URL
$ git checkout -b   # Creates a new branch and switches to it
$ git branch  # Lists all the branches in the repository
$ git log   # Displays the commit history for a specific branch
$ git cherry-pick   # Applies a specific commit from another branch to the current branch
$ git merge   # Merges another branch into the current branch (not a linear history)
$ git rebase   # Reapplies commits on top of the current branch for a linear history
